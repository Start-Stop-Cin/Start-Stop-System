# Requirements Documentation

This folder contains the requirements documentation produced across different phases of the Start-Stop System project.

## Structure

Each version folder includes:
- The **requirements specification document**
- The corresponding **requirements change report**

### Versions

- **V1.0**
  - Requirements baseline created after the elicitation and negotiation phase
  - Includes the initial system and software requirements specification

- **V2.0**
  - Updated requirements after the modeling phase (Simulink / Stateflow)
  - Includes refinements derived from system modeling and behavior validation
  - Contains a change report describing updates from V1.0 → V2.0

- **V3.0**
  - Final version with implementation-driven refinements
  - Includes additions such as:
    - SafeStop behavior
    - CO2 estimation reset logic
    - Fuel Estimation functionality
  - Contains a change report describing updates from V2.0 → V3.0

## Purpose

These documents provide traceability of requirement evolution throughout the project lifecycle, ensuring consistency between specification, modeling, and implementation.

## Notes

- Most requirement changes occurred during early phases (elicitation and scope definition)
- Later versions mainly include refinements identified during modeling and implementation phases
